# Codex 模型与 Skill 桥接路线图

## 当前版本

当前工作台是本地静态页面。

它能做：

- 和你进行前端对话
- 判断应该使用哪个助手
- 组合相关 skill 和工作流
- 生成可交给 Codex 执行的完整任务指令
- 打开相关网页、配置或本地应用入口

它不能直接做：

- 在网页里直接调用 Codex 模型
- 在网页里直接读取或执行 Codex skill
- 在网页里直接操作本机文件系统
- 在网页里直接启动剪映或其他 exe

这些动作需要由 Codex 会话、本地脚本、浏览器插件能力或后端服务完成。

## 第二阶段：本地桥接服务

如果要让工作台按钮直接触发执行，需要一个本地桥接服务。

可能架构：

```text
工作台网页
  -> 本地 HTTP 服务
  -> Codex / OpenAI API / 本机脚本
  -> Office、AnySearch、剪映、平台页面
```

需要补齐：

- 本地服务端口
- API Key 安全存储
- 请求权限确认
- 文件读写白名单
- 日志记录
- 错误回滚

## 第三阶段：真正的多助手执行

每个助手可以变成独立执行器：

- Office Executor
- News Executor
- Trading Alert Executor
- Inquiry Executor
- Editing Executor

每个执行器都有：

- 输入规范
- 可用 skill
- 可用工具
- 安全边界
- 输出文件位置

## 安全原则

- 不在网页里明文保存 API Key
- 不处理账号密码
- 金融交易不无人值守实盘
- 任何真实资金操作必须人工确认
- 所有外部信息保留来源

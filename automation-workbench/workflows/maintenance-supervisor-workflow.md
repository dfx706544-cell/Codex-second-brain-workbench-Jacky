# 维护助手 / 运行监管工作流

目标：确保第二大脑自动化工作台、Codex 协同、平台接入、云端任务、助手能力、知识库、历史记录和 API 费用提醒长期稳定可用。

## 默认执行原则

1. 先后台，后前台：优先用 browser、chrome、playwright、网页搜索、API、导出文件和本地配置检查；后台无法完成时，再请求接管电脑并打开前台页面。
2. 登录停下：遇到登录、验证码、二次验证、支付密码、交易密码、账号安全确认时立即停下，让用户亲自完成。
3. 不做不可逆动作：不直接发送邮件/社交消息，不发布内容，不提交表单，不真实交易，不支付，不静默安装第三方 skill/plugin/software。
4. 证据优先：每个“可用/不可用”的结论要记录检查时间、检查方式、URL 或本地文件路径、观察结果和下一步动作。
5. 不编造：API 余额、平台数据、账号数据、云端状态无法核实时，标注“待授权/待核实”，不要猜金额或数据。

## 每日轻巡检

建议每天在信息简报后运行，重点检查：

- 队列：`automation-workbench/queue/tasks.json` 是否可读，最新任务是否能被执行口令覆盖。
- 桥接：共享队列服务和运行中心是否在线；如本地备份模式出现，提示使用 `open-workbench.ps1`。
- 平台链接：检查 `automation-workbench/config/settings.json` 里 enabled 平台 URL 是否能打开或至少能到达登录页。
- 输出：`outputs/` 是否可写，今日简报、业务反馈、邮件草稿是否生成。
- 数据库：`automation-workbench/data/knowledge-items.json`、`daily-briefs.json`、`business-feedback.json`、`task-history.json` 是否可读。
- API 费用：如配置了真实账单/余额来源，则检查余额；低于 50 元人民币时提醒充值。未配置时记录“余额监控未配置/待授权”。
- 成本同步：记录当日任务预计人民币成本、已核实 API/token 消耗、待核实账单、第三方平台订阅和本地/云端运行成本。

## 每周深巡检

建议每周运行一次，重点检查：

- 工作台页面：主要标签、助手列表、知识库、历史记录、每日交付、运行中心是否正常渲染。
- 平台接入：Kalodata、FastMoss、达秘 / TikClubs、飞书、微信、163 邮箱、TikTok、YouTube、Instagram、抖音、视频号、小红书、SEC、HKEXnews、GitHub、Google Scholar 等平台是否仍可打开。
- Codex 协同：复制队列执行口令是否包含工作区路径、队列路径、后端优先、前台接管、登录停下、输出目录和历史记录要求。
- 自动化：每日 8 点信息简报、业务反馈、知识库更新、每周自我迭代审计是否仍处于 ACTIVE；提示词是否包含安全边界。
- 云端同步：GitHub 私有仓库、公开模板仓库、GitHub Pages 或 Codex Cloud 任务是否与本地隐私边界一致。
- skills/plugins/MCP：核对工作台登记能力和当前 Codex 可用能力是否一致；列出候选更新，但安装前等待用户确认。
- 隐私：公开模板不得包含 Jacky 的邮箱、历史记录、outputs、账号平台数据、本地绝对路径或私有仓库信息。
- 运行与维护成本：同步 GitHub Actions/Codex Cloud、LLM/API/token、邮件发送、第三方平台订阅、本地开机运行和手动确认时间成本。

## 输出格式

保存到 `outputs/maintenance-report-YYYY-MM-DD.md`，并尽量追加到 `automation-workbench/data/task-history.json`。

报告应包含：

- 总体状态：可用 / 部分可用 / 需要处理。
- 已检查项目：每项包含检查时间、方式、结果、证据链接或路径。
- 平台接入表：平台、URL、是否打开、是否需要登录、是否需要用户动作。
- 自动化状态：每日简报、业务反馈、知识库更新、每周审计、API 费用提醒。
- 预计/实际成本：本次任务预计人民币成本、已核实金额、待核实金额、余额是否低于 50 元。
- 问题清单：按紧急度排序。
- 已自动修复：只记录安全、可逆、无需外部确认的修复。
- 待用户确认：登录、验证码、邮箱发送、社交外发、真实交易、支付、安装第三方代码、授权新的账单来源。

## 判定标准

- “已接入”：工作台配置中存在平台，URL 可打开，且能到达首页、登录页或授权后的页面。
- “已授权可读”：用户已登录，页面内容可见，且不需要绕过权限。
- “待登录”：平台可打开，但需要用户登录、验证码或二次验证。
- “待配置”：平台或 API 需要密钥、账单来源、导出文件或连接器，当前无法核实。
- “不可用”：URL 无法到达、服务报错、配置缺失、脚本失败或权限不足。

# 信息助手工作流

目标：统一处理飞书、微信、邮箱、社媒私信和工作平台消息，先整理，再生成草稿，最后等待用户确认是否发送。

## 输入

- 用户指定的平台和账号范围。
- 当前浏览器中用户已登录且可见的消息页面。
- `inputs/` 中用户导出的聊天、邮件或客户沟通记录。

## 步骤

1. 打开用户指定平台；如遇登录、验证码、二次验证，等待用户亲自完成。
2. 只读取可见消息，不读取密码、验证码、浏览器密码库或无授权数据。
3. 按紧急度、客户价值、是否需要回复、是否需要附件、是否需要后续跟进分类。
4. 为每条需要回复的信息生成草稿，说明目标、语气、风险和建议发送时间。
5. 把回复草稿、待确认事项和跟进清单保存到 `outputs/`。
6. 不直接发送消息；发送前必须让用户确认。

## 输出

- `outputs/inbox_reply_drafts.xlsx` 或按任务命名的表格。
- 必要时生成 Word 版沟通纪要。
- 最后回复中列出哪些草稿等待用户确认。

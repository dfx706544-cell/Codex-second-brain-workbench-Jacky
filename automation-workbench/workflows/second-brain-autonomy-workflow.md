# 第二大脑 v4 自主运行工作流

目标：把工作台从“本地任务驾驶舱”升级为“可被后台自动化持续喂养的数据中枢”。本工作流定义哪些事情可以自主执行，哪些事情必须等待用户确认，以及电脑关机后的真实运行边界。

## 三层架构

### 1. 本地工作台层

位置：`automation-workbench/`

职责：

- 展示任务中枢、知识库、历史记录、每日交付、个人画像。
- 保存本地 JSON 数据：`automation-workbench/data/`。
- 作为用户下达任务、查看结果、审阅来源和确认外发的界面。

边界：

- 电脑关机后不能运行。
- 本地网页和本地桥接服务都依赖当前电脑开机、Codex 可用、项目文件存在。

### 2. Codex 自动化层

职责：

- 按固定时间唤醒 Codex。
- 自动收集资讯、整理简报、准备业务反馈、生成邮件草稿。
- 在有本地工作区访问权限时，把结果写入 `outputs/` 和 `automation-workbench/data/`。

适合自主执行：

- 搜集公开资讯。
- 整理信息简报。
- 生成业务反馈建议。
- 更新知识库条目。
- 生成邮件草稿。
- 生成自我迭代建议清单。

必须确认后执行：

- 发送邮件。
- 发送微信、飞书、私信或社交平台回复。
- 上传文件、提交表单、发布内容。
- 安装第三方 skill、插件或软件。
- 真实金融交易、支付、下单。
- 使用账号密码、验证码、支付码、交易密码。

### 3. 云端或常久在线层

用途：实现“电脑关机后仍然运行”。

可选方案：

- Codex 云端/远程工作区：适合让 Codex 定时跑研究、写文件、整理交付。
- VPS：适合稳定运行脚本、数据库、邮件发送器、网页服务。
- NAS：适合个人本地长期在线存储和定时任务。
- 云函数：适合轻量定时抓取和推送。

必要能力：

- 加密保存凭证。
- 明确权限范围。
- 发送邮件前的安全策略。
- 运行日志和失败告警。
- 备份与恢复。

## 每日自主任务

每天 8 点默认执行：

1. 信息简报：金融、宏观、社会热点、AI、跨境电商、创作者平台、学术信息。
2. 业务反馈：账号复盘、达人沟通、内容表现、跨境电商机会、明日行动。
3. 知识库更新：把可追溯来源写入 `knowledge-items.json`。
4. 交付草稿：为 `jacky060911@163.com` 准备两封邮件草稿。

## 自我迭代任务

建议每周执行一次：

1. 检查工作台任务历史和失败记录。
2. 搜索可信的 Codex skills、插件和自动化脚本候选。
3. 输出候选清单：用途、来源、维护状态、风险、是否需要账号或网络。
4. 等待用户确认后再安装。
5. 安装后写入历史记录和 README。

## 数据写入规范

自动化结果应尽量写入：

- `outputs/`：完整简报、报表、邮件草稿和附件。
- `automation-workbench/data/knowledge-items.json`：来源明确的知识条目。
- `automation-workbench/data/daily-briefs.json`：每日信息简报索引。
- `automation-workbench/data/business-feedback.json`：业务反馈索引。
- `automation-workbench/data/task-history.json`：完成记录和失败记录。

## 安全原则

- 默认“可自主分析，不自主外发”。
- 默认“可生成建议，不直接替用户做不可逆动作”。
- 默认“不保存敏感凭证”。
- 所有真实外部动作都需要当次确认，除非未来单独配置了安全发送器和明确白名单。
